## 1. 프로젝트 개요

### 목적

"묵상대학"은 소규모 그룹이 매일 묵상 기록을 제출하고 출석·지각·결석을 관리하며, 규칙 위반 시 벌금을 자동 계산하는 **묵상 챌린지 관리 서비스**이다.

### 핵심 가치

| 항목 | 설명 |
|------|------|
| 대상 사용자 | 정기적으로 묵상 챌린지를 함께 진행하는 소규모 그룹(스터디, 공동체 등) |
| 핵심 문제 해결 | 묵상 제출 여부와 제출 시각을 기준으로 출석/지각을 자동 판별하고, 누적 벌금을 간편하게 계산 |
| 현재 범위 (MVP) | 묵상 제출 → 출석/지각 판별 + 벌금 계산기 |

## 3. 현재 구현된 기능 명세

### 3-1. 묵상 제출 및 출석 판별

**개요**
사용자가 묵상 내용을 텍스트로 입력하고 제출하면, 제출 시각과 설정된 마감 시각을 비교하여 출석 또는 지각으로 판정한다.

**처리 흐름**

```
사용자가 묵상 내용 입력
  → 제출 버튼 클릭
  → 클라이언트에서 현재 시각(HH:MM) 자동 수집
  → POST /api/posts 호출
  → 서버에서 submission_time vs deadline_time 비교
  → 결과 반환 (status: "present" | "late")
  → 화면에 출석 상태 배지 표시
```

**판정 기준**

| 조건 | 결과 |
|------|------|
| 제출 시각 <= 마감 시각 | 출석 (`present`) |
| 제출 시각 > 마감 시각 | 지각 (`late`) |

**현재 고정값 (하드코딩)**
- 사용자 이메일: `user@meditation.uni` (로그인 기능 미구현으로 고정)
- 마감 시각 기본값: `06:00`

**UI 상태 표시**

| 상태 | 배지 텍스트 | 배지 색상 |
|------|------------|----------|
| 미제출 | "미제출" | 회색 |
| 출석 | "출석" | 초록색 |
| 지각 | "지각" | 노란색 |

---

### 3-2. 벌금 계산

**개요**
지각 횟수와 결석 횟수를 수동으로 입력하면 규칙에 따라 총 벌금을 계산한다.

**계산 로직**

```
환산 결석 = (지각 횟수 × 0.5) + 결석 횟수

if 환산 결석 < 4:
    총 벌금 = 환산 결석 × 3,000원

if 환산 결석 >= 4:
    총 벌금 = 30,000원 (상한 적용)
```

**벌금 기준표 (화면 내 표시)**

| 조건 | 금액 |
|------|------|
| 지각 1회 | 결석 0.5회로 환산 |
| 환산 결석 4회 미만 | 회당 3,000원 |
| 환산 결석 4회 이상 | 30,000원 (상한) |

## 4. 데이터 모델

> 현재 데이터베이스가 없으므로 API 요청/응답 스키마가 사실상 데이터 모델 역할을 한다.

### PostCreate (묵상 제출 요청)

| 필드 | 타입 | 설명 | 비고 |
|------|------|------|------|
| `user_email` | string | 사용자 이메일 | 현재 하드코딩 고정값 |
| `content` | string | 묵상 본문 텍스트 | |
| `submission_time` | string | 제출 시각 (HH:MM) | 클라이언트에서 자동 수집 |
| `deadline_time` | string | 마감 시각 (HH:MM) | 사용자 입력, 기본값 06:00 |

### PostResponse (묵상 제출 응답)

| 필드 | 타입 | 설명 |
|------|------|------|
| `status` | string | `"present"` 또는 `"late"` |
| `message` | string | 사람이 읽는 결과 메시지 |
| `submission_time` | string | 에코된 제출 시각 |
| `deadline_time` | string | 에코된 마감 시각 |

### FineRequest (벌금 계산 요청)

| 필드 | 타입 | 설명 |
|------|------|------|
| `late_count` | float | 지각 횟수 |
| `absent_count` | float | 결석 횟수 |

### FineResponse (벌금 계산 응답)

| 필드 | 타입 | 설명 |
|------|------|------|
| `total_fine` | int | 총 벌금액 (원) |
| `converted_absences` | float | 환산 결석 횟수 |
| `detail` | string | 계산 상세 내역 문자열 |

## 5. API 명세

### 공통 사항

- Base URL: `http://localhost:8000`
- Content-Type: `application/json`
- CORS 허용 출처: `http://localhost:5173`

---

### 5-1. 묵상 제출

```
POST /api/posts
```

**Request Body**

```json
{
  "user_email": "user@meditation.uni",
  "content": "오늘 묵상한 내용...",
  "submission_time": "05:30",
  "deadline_time": "06:00"
}
```

**Response 200**

```json
{
  "status": "present",
  "message": "출석이 확인되었습니다. (05:30 제출)",
  "submission_time": "05:30",
  "deadline_time": "06:00"
}
```

**Response 200 (지각 케이스)**

```json
{
  "status": "late",
  "message": "지각 처리되었습니다. (07:00 제출, 마감 06:00)",
  "submission_time": "07:00",
  "deadline_time": "06:00"
}
```

**에러 케이스**

| 상황 | HTTP 상태 | 비고 |
|------|-----------|------|
| 필수 필드 누락 / 타입 오류 | 422 Unprocessable Entity | Pydantic 자동 처리 |
| 시각 포맷 오류 (HH:MM 불일치) | 500 Internal Server Error | 현재 예외 처리 미구현 |

---

### 5-2. 벌금 계산

```
POST /api/fines/calculate
```

**Request Body**

```json
{
  "late_count": 2,
  "absent_count": 1
}
```

**Response 200**

```json
{
  "total_fine": 6000,
  "converted_absences": 2.0,
  "detail": "환산 결석 2.0회 × 3,000원 = 6,000원"
}
```

**Response 200 (상한 적용 케이스)**

```json
{
  "total_fine": 30000,
  "converted_absences": 5.0,
  "detail": "환산 결석 5.0회 → 상한 벌금 30,000원 적용"
}
```

**에러 케이스**

| 상황 | HTTP 상태 | 비고 |
|------|-----------|------|
| 필수 필드 누락 / 타입 오류 | 422 Unprocessable Entity | Pydantic 자동 처리 |

## 7. 미구현 / 개선 필요 기능

### 7-1. 구조적 한계 (Must Fix)

| 항목 | 현재 상태 | 문제점 |
|------|-----------|--------|
| 데이터 영속성 | 없음 (인메모리) | 서버 재시작 시 모든 데이터 소멸, 제출 이력 저장 불가 |
| 사용자 인증 | 없음 | 이메일이 `user@meditation.uni`로 하드코딩, 멀티 사용자 불가 |

| 시각 포맷 유효성 검사 | 미구현 | `submission_time`이 `HH:MM` 형식이 아닐 경우 500 오류 발생 |

### 7-2. 기능 누락 (Nice to Have)

| 항목 | 설명 |
|------|------|
